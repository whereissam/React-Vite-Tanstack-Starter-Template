# Security Policy

## Supported Versions

This project is a starter template. Security updates are applied to the latest
release on the `main` branch only.

| Version | Supported          |
| ------- | ------------------ |
| `main`  | :white_check_mark: |
| older   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

- **Do not** open a public GitHub issue for security problems.
- Email the maintainer at **tools@gaib.ai** with:
  - A description of the vulnerability and its potential impact.
  - Steps to reproduce (proof of concept, affected files, or routes).
  - Any suggested remediation, if known.

Alternatively, use GitHub's [private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability)
via the **Security** tab of this repository.

## Response Expectations

- **Acknowledgement:** within 3 business days of your report.
- **Status update:** within 7 business days, including a severity assessment.
- **Fix & disclosure:** coordinated once a patch is available. We will credit
  reporters who wish to be acknowledged.

## Scope

Because this is a template, please note:

- Dependency vulnerabilities should generally be reported upstream to the
  respective package maintainers. Run `bun audit` to check installed packages.
- Issues in example/demo code are in scope if they could mislead users into
  insecure patterns when adopting the template.

Thank you for helping keep this project and its users safe.
